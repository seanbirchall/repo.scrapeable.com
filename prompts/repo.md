# R Client Package Standards

Everything here is derived from building sec.gov, census.gov, and fred.stlouisfed.org. Follow these patterns for every new client.

## Package naming

Package name = the domain you're hitting. Dots are valid R package names and prevent CRAN collisions.

```
sec.gov
census.gov
fred.stlouisfed.org
clinicaltrials.gov
dol.gov
finra.org
```

## Dependencies

Every client uses the same core stack:

| Package | Purpose | Always? |
|---------|---------|---------|
| httr2 | HTTP requests | Yes |
| jsonlite | JSON parsing | Yes |
| dplyr | Data manipulation | Yes |
| tidyr | Reshaping (when needed) | Yes |
| tibble | Return type | Yes |
| xml2 | HTML/XML parsing | When scraping HTML or parsing XML |
| readxl | Excel files | Edge case only |
| pdftools | PDF text extraction | Edge case only |

Never add rvest, data.table, readr, httr (v1), or purrr. The core five (httr2, jsonlite, dplyr, tidyr, tibble) cover everything. Use edge case packages only when totally necessary.

## File structure

### Client file (clients/{domain}.R)

Self-contained, runs standalone with `source()`. This is the development file — iterate here first, package later.

```r
# {domain}.R
# Self-contained {Name} client.
# All public functions return tibbles.
#
# Dependencies: httr2, jsonlite, dplyr, tidyr, tibble
# Auth: {describe auth pattern}

library(dplyr, warn.conflicts = FALSE)
library(tidyr)
library(tibble)
```

### Package (packages/r/{domain}/)

```
packages/r/{domain}/
  DESCRIPTION
  LICENSE
  NAMESPACE          # hand-maintained, not roxygen-generated
  R/
    utils.R          # private helpers, schemas, fetch functions
    {domain}.R       # public functions with roxygen comments
  inst/
    source/{domain}.R  # copy of the standalone client (for context fn)
  man/               # generated by roxygen2::roxygenise()
```

### webR package (packages/webr/)

CRAN-like repo structure. All packages share one repo.

```
packages/webr/
  src/contrib/
    {domain}_0.1.0.tar.gz
    PACKAGES, PACKAGES.gz, PACKAGES.rds
  bin/emscripten/contrib/4.5/
    {domain}_0.1.0.tgz
    PACKAGES, PACKAGES.gz, PACKAGES.rds
```

Build with Docker:
```bash
docker run --rm -v ./packages/r:/input -v ./packages/webr:/output ghcr.io/r-wasm/webr:main bash -c '
  R --vanilla -e "install.packages(c(\"httr2\",\"jsonlite\",\"dplyr\",\"tidyr\",\"tibble\"), repos=\"https://cloud.r-project.org\")"
  mkdir -p /tmp/lib
  R CMD INSTALL /input/{domain}_0.1.0.tar.gz --library=/tmp/lib
  R_VER=$(R --version | head -1 | grep -oP "\d+\.\d+")
  cp /input/{domain}_0.1.0.tar.gz /output/src/contrib/
  cd /tmp/lib && tar czf /output/bin/emscripten/contrib/${R_VER}/{domain}_0.1.0.tgz {domain}
  cd /output/src/contrib && R --vanilla -e "tools::write_PACKAGES(\".\", type=\"source\")"
  cd /output/bin/emscripten/contrib/${R_VER} && R --vanilla -e "tools::write_PACKAGES(\".\", type=\"mac.binary\")"
'
```

## Naming conventions

### Function names: `{prefix}_{noun}()` or `{prefix}_{noun}_{modifier}()`

The prefix is a short, unique abbreviation of the domain:

| Domain | Prefix | Examples |
|--------|--------|---------|
| sec.gov | `sec_` | `sec_filings`, `sec_facts`, `sec_filing_xbrl` |
| census.gov | `census_` | `census_acs5`, `census_get`, `census_search` |
| fred.stlouisfed.org | `fred_` | `fred_series`, `fred_search`, `fred_releases` |
| dol.gov | `dol_` | |
| finra.org | `finra_` | |
| clinicaltrials.gov | `ct_` | |

### Patterns that emerge

- `{prefix}_search(query, ...)` — keyword search
- `{prefix}_get(...)` — core data fetching engine (if the API is uniform)
- `{prefix}_{noun}_bulk(...)` — batch/bulk version with progress messages
- `{prefix}_context()` — LLM context injection (every package)

### api_key placement

Always the last required positional argument or an optional keyword argument:

```r
# Required key (FRED, FMP)
fred_series(code, api_key)

# Optional key (Census — works without, just rate-limited)
census_acs5(year, variables, for_geo, in_geo = NULL, key = NULL)

# No key (SEC — just user-agent)
sec_filings(cik)
```

## Return values

### Always return tibbles

Every public function returns a tibble. No lists, no character vectors, no raw JSON. If a function downloads a file, return a tibble with path/metadata columns.

### Typed columns

Define schemas (typed empty tibbles) at the top of the file. Use them for early returns and as documentation of the return contract.

```r
.schema_series <- tibble(
  date = as.Date(character()), value = numeric(), code = character()
)
```

Cast types explicitly in the function body:

```r
df |> mutate(
  date = as.Date(date),
  value = suppressWarnings(as.numeric(value)),
  cik = as.integer(cik),
  across(any_of(c("filingdate", "reportdate")), as.Date)
)
```

### When NOT to auto-type

If the source data uses sentinel values for missing/suppressed data (Census uses "-", "N", "(X)"), return character and let the caller decide. Document this clearly.

## Private utilities

### .fetch / .fetch_json / .fetch_html / .fetch_xml

Every client defines these identically. They're 3-5 lines each. No shared helper file — each client is self-contained.

```r
.fetch <- function(url, ext = ".json") {
  tmp <- tempfile(fileext = ext)
  httr2::request(url) |>
    httr2::req_headers(`User-Agent` = .ua) |>
    httr2::req_perform(path = tmp)
  tmp
}

.fetch_json <- function(url) jsonlite::fromJSON(.fetch(url))
.fetch_html <- function(url) xml2::read_html(.fetch(url, ".html"))
.fetch_xml  <- function(url) xml2::read_xml(.fetch(url, ".xml"))
```

### URL builders

For APIs with a consistent pattern, define a URL builder:

```r
# FRED: all endpoints share same pattern
.fred_url <- function(endpoint, api_key, ...) {
  params <- list(...)
  params <- params[!vapply(params, is.null, logical(1))]
  query <- paste(names(params), params, sep = "=", collapse = "&")
  sprintf("%s/%s?%s&api_key=%s&file_type=json", .fred_base, endpoint, query, api_key)
}
```

### Session cache

Use a private environment for caching expensive lookups (entity lists, variable catalogs):

```r
.{prefix}_cache <- new.env(parent = emptyenv())
```

Cache on first access, return from cache on subsequent calls.

## S3 objects (complex APIs only)

For APIs where a single entity (filing, dataset) has multiple views/parsers, use an S3 object as the foundation:

```r
sec_filing(accession)     # → S3 "sec_filing" object (tibble with attrs)
sec_filing_xbrl(filing)   # → parser accepts the object
sec_filing_table(filing)  # → another parser, same object
```

The S3 object pattern:
- Returns a tibble (the file inventory / metadata)
- Carries metadata as attributes (accession, cik, entity, form, period, etc.)
- Custom `print()` method shows a rich summary THEN the tibble data via `NextMethod()`
- Sub-parsers accept the object OR a raw identifier (fetch internally if needed)
- `.tag_result()` copies attributes from the filing object to parser results

Don't use S3 for simple APIs (FRED, FMP). It adds complexity without benefit when every function is just `endpoint → tibble`.

## {prefix}_context() function

Every package must include a `{prefix}_context()` function that outputs LLM-friendly documentation. The format:

```
# {package} - {Description} for R
# Dependencies: {list}
# Auth: {describe}
# {Important notes about the data/API}
#
# {Common codes/variables/endpoints if applicable}
#
# == Functions ==
#
# {fn_name}({args})
#   {One-line description}
#   Run `{fn_name}` to view source or `?{fn_name}` for help.
#
# == Quick examples ==
#
# {runnable example}
```

The "Run `fn_name` to view source" line (no parens) tells the LLM how to get more info without executing the function.

## print methods

### S3 objects: rich header + tibble

```r
print.sec_filing <- function(x, ...) {
  # Rich summary header
  cat(strrep("\u2500", 70), "\n")
  cat("Entity name\n")
  cat("Key metadata on one line\n")
  cat(strrep("\u2500", 70), "\n")
  # Categorized inventory / summary stats
  # Available actions / parsers
  cat("\n")
  NextMethod()  # prints the tibble
  invisible(x)
}
```

### Regular tibbles: just return them

No custom print for regular function returns. Tibble's default print (10 rows, column types) is what we want.

## Testing

### Test file: tests/test-{domain}.R

Run with `Rscript tests/test-{domain}.R`. Output to `tests/{domain}-output.txt` for inspection.

Test pattern:
```r
source("clients/{domain}.R")

run <- function(name, expr) {
  cat(sprintf("\n--- %s ---\n", name))
  result <- tryCatch(expr, error = function(e) e)
  if (inherits(result, "error")) {
    cat("FAIL:", conditionMessage(result), "\n")
  } else {
    cat("OK\n")
    if (is.data.frame(result)) print(head(result, 6))
  }
  invisible(result)
}
```

### What to test

- Every public function with real API calls
- Schema returns (empty tibble has correct columns/types)
- Column types on real data (Date, numeric, integer, character)
- Edge cases: empty results, missing data, pagination

## Common patterns

### Bulk with progress

```r
fred_series_bulk <- function(codes, api_key, sleep = 0.5) {
  n <- length(codes)
  results <- lapply(seq_along(codes), function(i) {
    if (i > 1) Sys.sleep(sleep)
    message(sprintf("[%d/%d] %s", i, n, codes[i]))
    tryCatch(fred_series(codes[i], api_key), error = function(e) {
      message("  Failed: ", e$message)
      NULL
    })
  })
  bind_rows(results)
}
```

### Scraping HTML tables

```r
.html_tables <- function(doc, xpath = ".//table") {
  nodes <- xml2::xml_find_all(doc, xpath)
  lapply(nodes, function(tbl) {
    rows <- xml2::xml_find_all(tbl, ".//tr")
    if (length(rows) == 0) return(tibble())
    cells <- lapply(rows, function(r)
      xml2::xml_text(xml2::xml_find_all(r, ".//td|.//th"), trim = TRUE))
    max_cols <- max(lengths(cells), 0L)
    if (max_cols == 0) return(tibble())
    mat <- do.call(rbind, lapply(cells, function(r) { length(r) <- max_cols; r }))
    df <- as_tibble(as.data.frame(mat, stringsAsFactors = FALSE))
    if (nrow(df) <= 1) return(df)
    hdr <- as.character(df[1, ])
    hdr[is.na(hdr) | hdr == ""] <- paste0("X", which(is.na(hdr) | hdr == ""))
    names(df) <- make.unique(hdr)
    df[-1, , drop = FALSE]
  })
}
```

### Geography helpers (Census pattern)

When an API uses hierarchical geography codes, provide type-safe constructors:

```r
census_geo_tract <- function(state, county, code = "*") {
  list(for_geo = paste0("tract:", code),
       in_geo = paste0("state:", state, " county:", county))
}
```

The core `_get()` function accepts these lists:

```r
if (is.list(for_geo) && !is.null(for_geo$for_geo)) {
  in_geo <- in_geo %||% for_geo$in_geo
  for_geo <- for_geo$for_geo
}
```

## Packaging checklist

1. Develop in `clients/{domain}.R` — test with `source()`
2. Create `packages/r/{domain}/` skeleton (DESCRIPTION, LICENSE, NAMESPACE)
3. Split source into `R/utils.R` (private) and `R/{domain}.R` (public + roxygen)
4. Copy source to `inst/source/{domain}.R`
5. Run `roxygen2::roxygenise()` to generate man pages
6. `R CMD build` → `R CMD check` → `R CMD INSTALL`
7. Docker build for webR → update `PACKAGES` indexes
8. Upload `packages/webr/` contents to `repo.scrapeable.com`

## Client complexity tiers

### Simple (FRED, FMP, Tiingo, ClinicalTrials)

- JSON API with key
- `{prefix}_{noun}(args, key) → tibble`
- No S3 objects
- ~200-400 lines

### Complex (SEC, Census)

- Multiple data sources, file types, or nested resources
- S3 foundation object with sub-parsers
- Discovery/search/metadata functions
- Geography or form-specific routing
- ~600-1200 lines
